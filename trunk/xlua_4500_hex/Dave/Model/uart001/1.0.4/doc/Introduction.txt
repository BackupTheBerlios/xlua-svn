\page UART001_intropage Introduction
<p> A universal asynchronous receiver/transmitter(UART), 
is a type of "asynchronous receiver/transmitter", a piece of computer hardware that 
translates data between parallel and serial forms.</p>

A UART usually contains the following components:
<ul>
<li>	a clock generator, usually a multiple of the bit rate to allow sampling in the middle of a bit period. </li>
<li>	input and output shift registers </li>
<li>	transmit/receive control </li>
<li>	read/write control logic </li>
<li>	transmit/receive buffers (optional) </li>
<li>	First-in, first-out (FIFO) buffer memory (optional) </li>
</ul>

<p>USIC_UART_UART001 App provides non reentrant API to configure the  USIC channel and 
Read/Write APIs/Macros which can be used for achieving a non RTOS use case. 
Communication may be "full duplex" (both send and receive at the same time) or "half duplex" 
(devices take turns transmitting and receiving) or "loop back mode" (Output signal is connected to the 
receiver input of same UART channel).</p>

\section UART001_scopesec Scope
App provides non reentrant APIs to achieve  a UART communication usecase using a USIC channel.  

\section UART001_abbreviation Abbreviation
<OL>
<LI>UART:Universal Asynchronous Receiver-Transmitter</LI>
<LI>USIC: Universal Serial Interface Channel</LI>

<LI>GUI: Graphical User Interface</LI>

</OL>


\section UART001_reference  Reference

@ingroup UART001_overview

\page UART001_overviewpage Overview
USIC_UART_UART001 App provides non re-entrant APIs which allows a higher level App to reserve a USIC channel 
(configured in UART protocol mode of operation) and use channel for data transfer/receive operations. The App 
is designed to always use the USIC FIFO, with UI configurations
 provided to configure the FIFO Size and Trigger Limits. The App also provides UI to configure UART parameters
 such as Baud Rate, Word Length, Stop Bit and Parity.(Note:For XMC4500, Parity mode cannot be used, see XMC4500 Errata Sheet Rel. 1.0, USIC_AI.007 Table 11)
DMA mode is not used by the App to simplify the design. However this feature is provided in UART002  App.
The App supports the below modes of operation.

<OL>
<LI>Full Duplex</LI>
<LI>Half Duplex</LI>
<LI>Loop Back Mode</LI>
</OL>

<b>Full Duplex</b>
<p>In Full Duplex communication both transmission and reception can be done simultaneously. To achieve this 
usecase independent communication line is needed for each transfer direction. UART001 App consumes two IO002 App to 
achieve full duplex communication. User can do manual pin assignment for each IO002 App based on the 
connectivity requirement.</p>

<b>Half Duplex</b>
<p>In Half Duplex communication a single communication line is shared for both transmission and reception. In this
 mode UART001 App consumes a single IO002 App. Half duplex mode can be selected if user wants
multi-transmitter communication. In this case, user has to take care that only one transmitter is active at 
a time.

Note: Collision detection is not possible for the XMC45xx/XMC44xx series due hardware limitation.
</p>

<b>Loop Back Mode</b>
<p>In this mode transmit pin of the allocated USIC channel is connected to receive pin of the same channel 
<b>internally</b>. Data transmitted by the transmitter can be received by the receiver as if it would have been 
sent by other communication partner. In this mode no IO002 App is consumed.</p>

 
The App provides below Macros which provide register base address of mapped  USIC Channels. This allows the 
user to achieve programming at register bit level in use cases which cannot be achieved with UART001 APIs. Below 
macros are generated in the file UART001_Conf.h.
<OL>
<LI><i><b>UART001_x_USIC</b></i> - Macro to access Mapped USIC Module Base address of UART001/x App instance.
<LI><i><b>UART001_x_USIC_CH</b></i> - Macro to access Mapped USIC Channel Base address of UART001/x App instance.  	
</OL>
@}
