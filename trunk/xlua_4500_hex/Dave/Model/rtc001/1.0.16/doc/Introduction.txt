\page RTC001_intropage Introduction 
<p>
Real-time clock (RTC) is a clock that keeps track of the current time.
 The RTC is capable of running from an alternate source of power, so it 
 can continue to keep time while the primary source of power is off or 
 unavailable. The timer remains operational when the core domain is in 
 power-down. The kernel part of the RTC keeps running as long as the 
 hibernate domain is powered with an alternate supply source.
</p>
<div style="text-align: center;"><IMG SRC="..\rtc_new0.PNG" border=2 bordercolor=#336699 title="RTC Block Diagram"></div>

<p>
RTC001  App  provides   APIs to  Enable/Disable the RTC module and also 
provides timer & alarm interfaces for other Apps. The Alarm and Periodic Timer 
Interrupts can be enabled from the App UI.
</p>

\section RTC001_scopesec Scope
App provides timer as well as alarm interfaces for top level Apps. 
The periodic timer events and alarm event may be used as wake up source for Hibernate mode.

\section Abbreviation
<OL>
<LI>API: Application Program Interface</LI>
<LI>App: DAVE3 Application</LI>
<LI>GUI: Graphical User Interface</LI>
<LI>SW: Software</LI>
<LI>RTC: Real Time Clock</LI>
</OL>

\section Reference
Device Reference Manual.

\section Limitation
NA

\page RTC001_overviewpage Overview

The use cases of RTC001 App APIs  are as below.
<p>
The user can enable/disable the RTC by calling RTC001_Enable() & RTC001_Disable() APIs.
 App allows the user to configure the current date and time through the UI provided or can be 
 configured at runtime by using RTC001_Clock_SetTime() & RTC001_Clock_GetTime() APIs. Also App provides
 user to configure alarm through UI or at run time using API RTC001_ConfigAlarm()
</p> 
<p>
The configurations done in hibernate mode include the enabling of alarm event, periodic second event,
periodic minute event, periodic hour event, periodic month event and periodic year event.
</p>
<p>
App also provides UI to configure Interrupt handler for both alarm and RTC Periodic Timer interrupts. 
</p>
<p>
The App provides option for the user to manage the trigger either using NVIC_SCU001 App or EXCP001 App.
In the case of NVIC_SCU001 App the user has to define the timer and alarm handlers and configure the same in RTC001 App.
While in the case of EXCP001 App, handling of NMI shall be enabled in EXCP001 app.
</p>
