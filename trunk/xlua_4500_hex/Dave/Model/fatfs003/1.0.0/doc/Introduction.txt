@ingroup FATFS003_intro
@{
\page FATFS003_intropage Introduction
FatFs is a software module used to organize a storage medium. It abstracts the physical layer interfaces
to the storage medium and provides the functionalities to manage user data more efficiently with files and
directories. Embedded file system library (EFSL) and FatFs are two popular and freely available FAT
libraries for developing small embedded systems. FatFs is a generic FAT file system module for small embedded systems that can be ported to any
underlying hardware. It has been architected with separate layers for hardware accesses
Some applications may directly use this App bypassing the Standard Library Wrapper App above it. 
User must mount the device first before doing any operations on the disk. Make File system api f_mkfs() takes
long time to create file system on the disk using driver code. This is due to the fact that FAT file system erases and writes single block at
a time.  FATFS003 App is used in RTOS environment.ID's is used as a synchronization object.Please refer the link below for more information on the FAT FS Api's.
<OL>
<LI>http://elm-chan.org/fsw/ff/00index_e.html</LI>
</OL>


\section FATFS003_scopesec Scope
FileSystem_FATFS_RTOS_FATFS003 App lies below the Standard Library App .FatFs module is completely separated from disk I/O layer. 
FatFs version R0.09 is used.It intergerates with Device Abstraction Layer to access the physical media in RTOS environment as shown below:

@ingroup FATFS003_filestructure File Structure
@{
\page FATFS003_filestructurepage File Structure 
Following files are generated : 
<OL>
<LI>ff.c:This is the third party source file containing all the function definations of FAT FS App. </LI>
<LI>FATFS003_Conf.c: This file defines the App's Init function.  </LI>
<LI>FATFS003.h: This file contains function prototype & data structure declarations.  </LI>
<LI>ff.h: This is the third party file containing FatFs configurations and declarations </LI>
<LI>ffconf.h: This is the FAT file system module configuration file. </LI>
<LI>syscall.c: This contains code of OS dependent controls for FatFs. </LI>
</OL>
